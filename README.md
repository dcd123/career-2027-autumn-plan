# 使用方式

## 核心原则

本仓库只要求每周维护一次。
平时只学习，不折腾系统。

GitHub 是成果归档，不是每日打卡负担。

## 平时怎么用

1. 打开 `WEEKLY_CURRENT.md`
2. 看本周主目标和任务清单
3. 按自己的节奏完成任务
4. 临时想法可以记在手机备忘录、Obsidian 或草稿纸
5. 不要求每天 commit
6. 不要求每天写正式笔记

## 周日怎么用

每周日统一维护：

1. 更新 `WEEKLY_CURRENT.md` 的完成情况
2. 整理一篇 `notes/weekly-notes/2026-Wxx.md`
3. 写一篇 `review/2026-Wxx-review.md`
4. 归档本周 `weekly/2026-Wxx.md`
5. 生成下周 `WEEKLY_CURRENT.md`
6. commit 一次

## 手机端怎么用

手机端只看：

`WEEKLY_CURRENT.md`

不要在手机端维护复杂目录。

## 电脑端怎么用

电脑端负责：

- 写代码
- 刷题
- 整理周笔记
- 周日复盘
- commit

## 最低保底原则

一周状态很差时，只做：

- 3 道算法题
- 1 篇短笔记
- 1 次代码或笔记整理
- 1 次周复盘

只要不断线，就不算失败。
