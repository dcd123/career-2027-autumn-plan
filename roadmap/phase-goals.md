# 阶段目标

## 阶段目标的使用方式

- 每 4 周检查一次当前阶段目标，不因单周失败推翻全局。
- 每个阶段只追求 3 到 5 个关键产出。
- 若连续两周低状态，优先保项目主线和算法最低频率。

## 阶段 1 验收

时间：2026-06-22 至 2026-08-31

- 能手写反转链表、二分查找、有效括号、两数之和、合并两个有序链表。
- 能用 Java 写出集合、异常、IO、泛型、基础线程示例。
- 能用 Git 完成一次 feature 分支开发并合并。
- 能用 Linux 命令定位日志中的错误行。
- 完成 Java 基础项目的第一个可运行版本。

## 阶段 2 验收

时间：2026-09-01 至 2026-10-31

- 能完成用户注册登录接口、RBAC 权限接口和接口文档。
- 能解释 MySQL 索引失效、事务隔离级别和慢 SQL 排查流程。
- 能解释 TCP 三次握手、四次挥手、HTTPS 基本流程。
- 能用 Docker 启动 MySQL/Redis 并运行项目 1。

## 阶段 3 验收

时间：2026-11-01 至 2026-12-31

- 能完成缓存查询、缓存更新、缓存穿透/击穿/雪崩处理实验。
- 能解释 JVM 内存区域、GC Roots、常见垃圾回收器。
- 能解释进程线程、死锁、虚拟内存、IO 多路复用基础。
- 能完成一次接口压测并记录优化前后数据。

## 阶段 4 验收

时间：2027-01-01 至 2027-03-15

- 能用 MQ 完成异步任务，并处理重复消费和失败重试。
- 能用 Docker Compose 启动项目依赖。
- 能用 Nginx 完成反向代理配置。
- 能讲清楚项目 1 和项目 2 的架构、瓶颈和改进路线。

## 阶段 5 验收

时间：2027-03-16 至 2027-05-31

- 能完成文档上传、解析、切分、向量化、检索、重排和引用溯源。
- 能构造一个问答评测集，并记录召回质量。
- 能实现 Agent 工具注册、任务状态机、日志追踪和人工确认。
- 能解释 RAG 与 Agent 项目的工程边界和失败场景。

## 阶段 6 验收

时间：2027-06-01 至 2027-08-31

- 能在 45 分钟内完成 2 道中等算法题并讲清思路。
- 能完成至少 8 次模拟面试记录。
- 简历项目经历有数据、有取舍、有可验证结果。
- 每个项目至少准备 20 个面试追问答案。
