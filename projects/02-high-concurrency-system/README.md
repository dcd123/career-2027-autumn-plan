# 项目 2：高并发系统项目

## 目标

做一个能展示缓存、异步削峰、限流、幂等、分布式锁和压测能力的后端项目。项目不追求业务复杂，重点展示高并发场景下的工程取舍。

## 必备功能

- Redis 缓存。
- MQ 异步削峰。
- 限流。
- 幂等。
- 分布式锁。
- 数据库索引优化。
- 压测报告。

## 建议场景

- 秒杀预约。
- 抢券系统。
- 热点内容浏览与点赞。
- 订单异步处理。

## 里程碑

1. 建立基础业务流程。
2. 加入 Redis 缓存并记录命中率。
3. 加入 MQ 异步处理。
4. 加入限流、幂等和分布式锁。
5. 做数据库索引优化。
6. 使用 JMeter/k6 做压测。
7. 输出优化前后对比报告。

## 面试证明材料

- 压测报告。
- 缓存策略说明。
- MQ 可靠性处理说明。
- 幂等和锁的失败场景说明。
